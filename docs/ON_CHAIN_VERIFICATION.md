# NEXUS Protocol - On-Chain Verification Guide

## Overview

After deploying the 5 NEXUS programs to devnet, this guide provides step-by-step instructions to verify:

1. **All programs are deployed and executable** on devnet
2. **CPI chains work correctly** (Layer 1 → Layer 2 → Layer 3 → Layer 4 → Layer 5)
3. **State transitions are correct** (accounts created, data modified)
4. **Compliance enforcement works** on-chain
5. **End-to-end flow completes** without errors

## Part 2: CPI Chain Verification

### 2.1 Create Test Entities (Layer 1)

Create entities through the Entity Registry program:

```bash
# Using Anchor client (TypeScript/JavaScript)

const { Connection, Keypair, PublicKey } = require("@solana/web3.js");
const { Program, AnchorProvider } = require("@project-serum/anchor");
const fs = require("fs");

// Connect to devnet
const connection = new Connection("https://api.devnet.solana.com", "confirmed");
const keypair = Keypair.fromSecretKey(
  Buffer.from(JSON.parse(fs.readFileSync(process.env.HOME + "/.config/solana/id.json")))
);
const provider = new AnchorProvider(connection, { publicKey: keypair.publicKey }, {});

// Load Entity Registry IDL
const entityRegistryId = new PublicKey("6fEr9VsnyCUdCPMHY7XYV6SFsw7td48aN9biM1UowzGh");
const idl = require("../target/idl/entity_registry.json"); // Generated by anchor build
const program = new Program(idl, entityRegistryId, provider);

// Register an entity
const entityId = Keypair.generate();
await program.methods
  .registerEntity(
    entityId,
    "Singapore",
    true, // kyc_verified
    1735689600, // kyc_expiry
    100_000_000, // single_transfer_limit
    500_000_000 // daily_limit
  )
  .accounts({
    entity: entityId.publicKey,
    signer: keypair.publicKey,
    systemProgram: "11111111111111111111111111111111",
  })
  .rpc();

console.log("✅ Entity registered:", entityId.publicKey.toString());
```

### 2.2 Create Pool (Layer 2)

Create a pool and add entities:

```bash
const pooling = new Program(poolingIdl, poolingEngineId, provider);

const poolId = Keypair.generate();
await pooling.methods
  .createPool(
    poolId,
    [entityId.publicKey], // entities
    1_000_000_000 // sweep_threshold
  )
  .accounts({
    pool: poolId.publicKey,
    admin: keypair.publicKey,
    systemProgram: "11111111111111111111111111111111",
  })
  .rpc();

console.log("✅ Pool created:", poolId.publicKey.toString());
```

### 2.3 Run Netting Cycle (Layer 2 → Layer 3 → Layer 4)

Trigger the netting algorithm:

```bash
await pooling.methods
  .runNettingCycle(poolId)
  .accounts({
    poolState: poolId.publicKey,
    poolAdmin: keypair.publicKey,
  })
  .rpc();

console.log("✅ Netting cycle executed");
```

**What happens:**

- Layer 2 reads entity positions
- Layer 2 calculates offsets
- Layer 3 compliance hook validates each transfer
- Layer 4 FX netting applies exchange rates
- Layer 5 sweep trigger creates loans if needed

## Part 5: Troubleshooting On-Chain Tests

### Issue: "Account not found"

**Cause:** Program hasn't been deployed yet or account wasn't created
**Solution:** Verify program is deployed first, then ensure transaction succeeded

### Issue: "Instruction failed due to invalid account"

**Cause:** Account is not the right type or owner
**Solution:** Check account derivation and ownership

### Issue: "Transaction timed out"

**Cause:** Devnet congestion or account initialization taking time
**Solution:** Increase RPC timeout or retry after a few seconds

### Issue: "Program returned error: custom program error: 0x1"

**Cause:** Custom program error from Anchor
**Solution:** Check program logs with `-vvv` flag:

```bash
solana logs <PROGRAM_ID> --url https://api.devnet.solana.com
```
